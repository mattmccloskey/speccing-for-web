Accessibility (a11y)

As from January 2010, all new EUROPA websites have to be created in compliancy with the Web Content Accessibility Guidelines 2.0, level AA.

Spec

Think not about pixel size, but how text size relates to the things around it.

Design semantically - This is an H1, not a 32px piece of text.

Spec from the top down - Start by defining know elements in the design (e.g. paragraph text, header text, links, buttons, etc.), then reference those, and make notes about exceptions if something doesn't play by the rules. (e.g. - this is header text BUT 120% normal size, or Blue instead of black). Once an exception is reused several times, make a top level definition for it.

Font sizes, as well as padding & margins, should be specced in rems, except for where it is known that size is relative to a specefic parent (e.g. super text and inline icons).
